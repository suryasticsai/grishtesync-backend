## ⚠️ CRITICAL RULES – VIOLATIONS WILL CAUSE BUILD FAILURE

### A. Requirements.txt
- **Spelling**: `gradio` (NOT `grado`, `gradio3`, `gradio4`).
- **No exact versions**: use `>=` (e.g., `gradio>=4.0.0`), never `==`.
- **Torch rule**: if you use `transformers`, **must** include `torch>=2.0.0`.
- **Dash rule**: include `dash>=2.14.0` and `dash-bootstrap-components>=2.0.0`.
- **Common dependencies**: `gradio`, `streamlit`, `flask`, `fastapi`, `django`, `transformers`, `torch`, `plotly`, `pandas`, `numpy`, `huggingface_hub`, `PyPDF2`, `openai`, `anthropic`.

### B. Gradio Rules
- **❌ Forbidden**: `allow_flagging`, `demo.queue()`.
- **✅ Required**: `gr.File(type="filepath")` for uploads, `demo.launch(server_name='0.0.0.0', server_port=7860)`.
- **Do**: keep core logic separate from UI.
- **Don't**: hardcode credentials; use environment variables.

### C. Streamlit Rules
- **Do**: use `st.session_state` to persist data, `@st.cache_data` for expensive operations, `st.secrets` for secrets.
- **Don't**: rely on regular variables for state; avoid deep nesting of `st.columns`.

### D. Flask Rules
- **Do**: use application factory pattern and blueprints for large apps, `jsonify()` for JSON responses.
- **Don't**: hardcode secrets; never check `request.method == 'POST'` without specifying `methods=['POST']` in route.

### E. FastAPI Rules
- **Do**: use Pydantic models for request/response validation, dependency injection, `async def` for I/O.
- **Don't**: use `...` as default for required fields; avoid blocking I/O in async routes.

### F. Django Rules
- **Do**: use ORM to prevent SQL injection, keep `DEBUG = False` in production, use `select_related()` / `prefetch_related()`.
- **Don't**: commit `SECRET_KEY` to version control; bypass CSRF protection for state‑changing requests.

### G. Transformers Library Rules
- **Do**: use `pipeline()` for standard tasks, batch processing, half‑precision (`torch.float16`), quantization with `bitsandbytes`.
- **Don't**: forget `torch` in requirements.txt; use outdated versions; omit `device` argument when a GPU is available.

### H. Plotly Dash Rules
- **Do**: use multi‑page structure, `patch` objects for partial updates, `dcc.Store` for client‑side state.
- **Don't**: use global variables for user state; create duplicate callbacks; rely on `block=True` without ensuring dash‑bootstrap‑components ≥2.0.0 – use `style={'width': '100%'}` for safety.

### I. Static HTML/CSS/JS Rules
- **Do**: include viewport meta tag, use semantic HTML5, appropriate input types.
- **Don't**: use inline event handlers (use `addEventListener`), inline CSS (use external file), commit secrets to static files.
